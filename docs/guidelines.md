Development Guidelines
===

James will maintain a collection of principles and mythos to guide development on this project.

* Any and all tech debt should be logged in TODO.md at the moment of discovery.
* Create a design document in /docs before starting a big sprint.
* Accurately report code coverage whenever a major feature is completed.
* Don't update the README.md until the sprint is complete.

# Anecdotes:

* I've never regretted creating another Maven module, even if it's only for one file.
* Great software engineering is all about giving users the highest degree of flexibility without ever lying or obfuscating.
